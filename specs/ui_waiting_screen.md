# UI Specification: Waiting Screen (等待畫面)

## 1. 概述
當使用者提交餐廳推薦請求後，系統會進入非同步處理流程。等待畫面需要提供清晰的進度反饋，減少使用者的焦慮感，並利用這段時間提供有趣的資訊。

## 2. 功能需求

### 2.1 輪詢機制 (Polling)
- **API**: `GET /v2/recommendations/status/{job_id}`
- **頻率**: 每 3 秒一次
- **超時**: 如果 120 秒後仍未完成，顯示超時提示。

### 2.2 進度顯示
後端會返回 `progress` (0-100) 和 `message`。前端應根據這些資訊更新 UI。

| Progress | 顯示訊息 | 動畫/圖示 |
|----------|----------|-----------|
| 0-10% | 準備中... | 準備餐具 |
| 10-30% | 正在搜尋餐廳資料... | 搜尋地圖 |
| 30-60% | 正在分析菜單... | 閱讀菜單 |
| 60-90% | 正在閱讀顧客評論... | 閱讀評論 |
| 90-99% | 正在為您量身打造推薦... | 廚師烹飪 |
| 100% | 完成！ | 上菜 |

### 2.3 趣味小知識 (Trivia)
在等待過程中，輪播顯示關於該餐廳類型或料理的小知識。
- **例子**: "你知道嗎？拉麵最早是從中國傳入日本的..."
- **頻率**: 每 5-10 秒切換一則。

## 3. 視覺設計 (Visual Design)

### 3.1 佈局
- **中央**: 大型進度環或進度條。
- **上方**: 當前狀態訊息 (e.g., "正在分析菜單...")。
- **下方**: 趣味小知識卡片。
- **背景**: 模糊的餐廳圖片或與料理相關的圖案。

### 3.2 錯誤處理
- 如果狀態為 `failed`：
  - 顯示錯誤訊息 (友善的用語，如 "哎呀，廚房有點忙不過來...")。
  - 提供 "重試" 按鈕。
  - 提供 "返回首頁" 按鈕。

## 4. 實作建議 (React/Next.js)
- 使用 `SWR` 或 `React Query` 進行輪詢。
- 使用 `framer-motion` 製作平滑的進度條動畫。
- 預先定義好 Trivia 資料庫 (JSON)，隨機選取顯示。
